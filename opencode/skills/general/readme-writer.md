# README Writer Skill

Use this skill when the user asks to create, update, simplify, or review a repository `README.md`.

## Goal

Create concise README documentation for human developers.

## Rules

- Keep the README short and practical.
- Link to detailed documentation instead of duplicating it.
- Do not copy agent-only rules into human documentation.
- Do not modify production code.

## Required Output

Return:

1. Files changed.
2. What changed.
3. Any follow-up documentation gaps.
